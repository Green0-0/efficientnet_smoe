4/5/2026
Made baseline_fast_sweep_log.py
This is a rework of the efficientnet training process for inaturalist that makes it finish training in a few hours instead of up to a week.
It uses optuna for hyperparameter sweeping and wandb for logging
- Specified batch size and lr for sweeping, where lr is a linear function of batch size
- Exact copy of efficientnet-b0 architecture (~5m params iirc)
- Swapped exp scheduler for cosine, due to reduced epoch count (350 -> 30)
- Uses inaturalist mini, which has 50 data points per class and 10k or so classes
- Does stratified splitting so that each class has an equal number of validation rows and there isn't class imbalance

4/12/26
Decided to do transfer learning on the EfficientNet-B0. Transfer learning is cut into two stages, one where the head is trained and one where the body+head is trained. All parts use a different LR.
Sweep is also done over epochs allocated to head, grad accum and weight decay. Dataset changed to iNat 2019 (oops) which is manually partitioned into 45/5/50 train/val/test for quicker training and no contamination.
Used sqrt LR scaling instead of linear LR scaling. Also cleaned up the scripts manually.

Epochs was dropped to 10 with a bigger split (81/9/10), and head epochs was allowed to be 0 such that it can be skipped.
